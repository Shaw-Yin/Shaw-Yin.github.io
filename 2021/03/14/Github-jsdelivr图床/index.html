Github+jsdelivr图床

2021-03-14
2021-03-14
Hexo

发现blog之前的几篇文章图片失效了，不管是sm.ms还是微博图床，都有失效的内容。对于一个图床来说，稳定性应该是首先要考虑的。Github免费提供有限的空间，对于blog图床来说已经够用。
唯一的缺点在于，国内访问GitHub非常的慢，最近还有时断时续无法连通的情况。我们可以使用cdn加速来解决这一问题，jsdelivr是一款原生支持github的开源免费cdn，用来加速存放在GitHub的静态内容十分合适。
此外借助picgo工具可以方便的上传、管理图片并自动转化成jsdelivr加速链接。

steps

  • 首先，新建一个repo用做图床
  • 其次，在GitHub/settings/developer settings/里申请一个personal access token
  • 然后，下载picgo，在图床里选在gh，填入对应的account/repo、repo的branch、刚刚申请的toker
  • 最后，在自定义域名里填上jsdelivr的加速链https://cdn.jsdelivr.net/gh/youraccount/repo

大功告成！

可以优化的点

  • 全站jsdelivr加速，这点有点麻烦，以后再说
  • 图片上传前压缩，这是个好方法，如果能把图片压缩整合进picgo就更好了
